# 回归测试场景清单

> 每次实现新功能后，必须回测以下场景，确保已有功能不被破坏。

## 空数据库场景

| # | 场景 | 预期结果 |
|---|------|----------|
| 1 | 首次启动（无 DB），打开首页 | 页面正常显示工具栏（扫描/清空/刷新按钮），不白屏 |
| 2 | DB 为空时调用 `/filters` | 返回空数组 `[]`，不返回 `null` |
| 3 | DB 为空时点击审片模式 | 按钮禁用或自动切回预览模式，不出现空白页 |
| 4 | DB 为空时点回收站 | 显示"回收站为空"，列表不报错 |

## 匹配场景

| # | 场景 | 预期结果 |
|---|------|----------|
| 5 | 同目录同名 ARW+JPG（如 `DSC01892.ARW` + `DSC01892.JPG`）| 配对成功，match_status=paired |
| 6 | 跨目录同名 ARW+JPG（如 `raw/DSC01892.ARW` + `jpg/DSC01892.JPG`）| 配对成功 |
| 7 | 同名但拍摄日期不同（不同天拍的同名文件）| 保持 orphan，不错误配对 |
| 8 | 不同名但拍摄日期相同 | 通过时间匹配配对成功 |
| 9 | 只有 RAW 无 JPG（如纯 ARW 目录）| 全部 orphan，不配对 |
